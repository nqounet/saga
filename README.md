# SAGA (Swift AVIF Graphic Assistant)

![SAGA Header](Resources/header.jpg)

SAGAは、macOS 13 Ventura以降のネイティブ機能（SwiftUI / Combine / AppKit）を活用し、ローカルフォルダ内のAVIF形式の画像ファイルをコミック閲覧に最適化されたレイアウトでブラウジングする、軽量なデスクトップアプリケーションです。

---

## 1. 動作環境
- **OS**: macOS 13.0 (Ventura) 以上
- **開発言語/ツール**: Swift 5.9 以上 (Xcode 15以上推奨)

---

## 2. 主な機能

- **非同期フォルダスキャン**: 
  指定されたフォルダ内から拡張子 `.avif`（大文字・小文字を区別しない）のファイルのみを抽出します。ファイル一覧の取得処理は非同期（Backgroundスレッド）で行われるため、大量のファイルが存在する場合でもUIがフリーズしません。
- **自然順ソート**: 
  抽出されたファイルは、ファイル名に基づいて自然順（例: `2.avif` が `10.avif` より前に来るよう `localizedStandardCompare(_:)` を使用）でソートされてインデックス化されます。
- **柔軟な表示モード**: 
  - **表示枚数**: 「1枚表示」または「2枚表示（見開き）」を切り替え可能。
  - **綴じ方向（ページ方向）**: 
    - `右開き (RTL)`: 日本のコミックスタイル。インデックスが若い画像が「右側」に配置されます。
    - `左開き (LTR)`: 洋書スタイル。インデックスが若い画像が「左側」に配置されます。
  - **1枚ずらし (isShifted)**: 
    2枚表示モード時に、偶奇の割り当てを調整するフラグ。
- **キーボードナビゲーション**: 
  左右の矢印キーでページを遷移できます。綴じ方向の設定（RTL/LTR）に応じて、ページ遷移時のキー挙動が動的に反転します。
- **非同期画像読み込みとメモリキャッシュ**: 
  `SagaImageLoader` が画像を非同期にロードし、`NSCache` を用いてメモリ上にキャッシュするため、滑らかで高速な描画パフォーマンスを発揮します。

## 3. 実装コア仕様・アルゴリズム

実装済みのソースコード（`SagaCore`）における主要ロジックの動作仕様は以下の通りです。

### 3.1 ポインタの自動調整 (`validatePointer`)
表示の基準となるインデックス（`pointer`）の値が変更された際、または表示設定が変更された際に自動的に呼び出され、不整合を防ぎます。
- **範囲制限**: `pointer` は常に `0` 〜 `maxIndex`（画像総数 - 1）の範囲内に制限されます（画像がない場合は `0` にリセットされます）。
- **2枚表示モード時の偶奇調整**:
  - **`isShifted` が `true`（有効）の場合**:
    - 画像が2枚以上あれば、`pointer` は**奇数** (`1, 3, 5...`) に自動調整され、最小値は `1` に制限されます。もし `pointer` が偶数の場合は、次の奇数（`pointer + 1 <= maxIndex` であれば `pointer + 1`、超える場合は `pointer - 1`）に補正されます。
    - *※これにより、`isShifted` 有効時はインデックス `0` (最初の画像) は表示されず、インデックス `1` 以降から2枚ずつペアで表示されます。*
  - **`isShifted` が `false`（無効）の場合**:
    - `pointer` は常に**偶数** (`0, 2, 4...`) に自動調整されます。奇数の場合は `pointer - 1` に補正されます。

### 3.2 画像マッピング・ロジック (`calculateDisplayIndices`)
現在の `pointer`、`displayCount`、および `pageDirection` から、左右のビューに描画すべき画像インデックス `(left, right)` を決定します。
- **1枚表示時 (`displayCount == 1`)**:
  - `pageDirection == .ltr` の場合: `(left: pointer, right: nil)`
  - `pageDirection == .rtl` の場合: `(left: nil, right: pointer)`
- **2枚表示時 (`displayCount == 2`)**:
  - 基準インデックス `first = pointer` と、次のインデックス `second = pointer + 1`（存在すれば）をペアとして割り当てます。
  - `pageDirection == .rtl` (右開き) の場合: `(left: second, right: first)` （若いインデックスが右側）
  - `pageDirection == .ltr` (左開き) の場合: `(left: first, right: second)` （若いインデックスが左側）

### 3.3 ページ遷移の計算
- **遷移幅（ステップ数）**:
  - 1枚表示モード時は `1` ページずつ移動。
  - 2枚表示モード時は `isShifted` のオン/オフに関わらず、通常通り `2` ページずつ移動します。
- **キー操作と遷移方向**:
  | ページ方向 | 押下キー | 内部ポインタの挙動 | ユーザーから見た効果 |
  | --- | --- | --- | --- |
  | **右開き (RTL)** | `左矢印キー (←)` | ポインタを増加させる (`+step`) | 左に進む（次ページへ） |
  | | `右矢印キー (→)` | ポインタを減少させる (`-step`) | 右に戻る（前ページへ） |
  | **左開き (LTR)** | `右矢印キー (→)` | ポインタを増加させる (`+step`) | 右に進む（次ページへ） |
  | | `左矢印キー (←)` | ポインタを減少させる (`-step`) | 左に戻る（前ページへ） |

## 4. プロジェクト構成

```
.
├── Package.swift            # Swift Package Manager 設定ファイル
├── spec.md                  # 設計仕様書（一部実装と異なる部分があります）
├── README.md                # 本ファイル
└── Sources/
    ├── SagaCore/            # アプリケーションのドメインロジック（UI非依存）
    │   ├── SagaCore.swift
    │   ├── SagaImageLoader.swift # 画像ロード & キャッシュ管理
    │   └── SagaViewerState.swift # 状態管理・各種アルゴリズム
    ├── Saga/                # メインアプリケーション（SwiftUI UI層）
    │   ├── main.swift       # エントリーポイント
    │   └── ContentView.swift # メイン画面レイアウト・キー監視
    └── SagaTests/           # コアロジックのテストコード
        └── main.swift
```

---

## 5. ビルドと実行方法

Swift Package Manager を使用して、ターミナルからビルドおよび起動が可能です。

### アプリケーションの起動
```bash
swift run Saga
```

### テストの実行
`SagaTests` ターゲットを実行することで、コアロジックの動作検証テストが行われます。
```bash
swift run SagaTests
```
